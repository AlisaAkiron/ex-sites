# Add a new ex-site

Each generator is a `SiteConfig` + an SVG map. The shared `DominanceTool` /
`DominanceMap` engine renders the map, handles region clicks → level picker,
tracks the score, persists to localStorage, and exports a PNG. Adding a site is
data + wiring, never engine changes. Mirror the existing `china-ex` site —
read these before starting so your new site matches conventions exactly:

- `src/sites/china-ex/config.ts` — the `SiteConfig` to copy
- `src/sites/china-ex/map.svg` — the SVG format to match
- `src/routes/china-ex.tsx` — the route to copy
- `src/features/dominance-map/types.ts` — the `SiteConfig` / `LevelDef` types
- `src/features/dominance-map/DominanceMap.tsx` — how the SVG is driven (read the
  region/level/score logic; it explains the gotchas below)

Pick a short kebab-case `id` (e.g. `foodie-ex`). It's used for the directory,
route (`/<id>`), storage key, and cover filename.

## 1. Get the source map

Clone/fetch the source project and find its working map. It's often inside an
`index.html` (a `<svg>…</svg>` block) or a standalone `.svg`. Identify:

- the region group (`<g id="…">`) and its child shapes (the clickable regions),
- the title text element and the score text element,
- the level definitions — colors **and** labels (often in a legend `<g>` and/or
  the picker markup),
- any watermark (QR code, the original site's domain) to drop.

## 2. Build `src/sites/<id>/map.svg`

Transform the source SVG into the project format (see `china-ex/map.svg`). The
transform is source-specific, so do it as targeted edits, not a fixed script.
Required changes:

- **Convert every region `<rect>` to a `<path>`.** This is the #1 gotcha:
  `DominanceMap` only sets the `level` attribute on — and only accepts clicks
  for — `<path>` children of the region group. A `<rect>` region silently won't
  color or respond. A rect maps to an identical path:
  `<rect x="X" y="Y" width="W" height="H"/>` → `<path d="MX,YhWvHh-Wz"/>`
  (keep the `id` and any other attributes). Legend/decorative rects don't matter.
- **Use the bundled font.** The export embeds only `ZCOOLKuaiLe`
  (`public/fonts/zcool-kuaile.ttf`). Replace any source `font-family` in the
  SVG's `<style>` with `'ZCOOLKuaiLe', sans-serif`.
- **Keep level fill rules** for `level="1".."5"` in the `<style>` (colors must
  match the `levels` you put in config).
- **Drop watermarks** (QR group, source domain text). Attribution lives in
  `config.credit`, not baked into the image.
- Keep the `<?xml …?>` prolog and `<svg … viewBox="0 0 W H">` wrapper.

Note the element ids you end up with — the title and score ids vary by source
(china-ex uses `标题`; foodie-ex kept the source's `title`). You'll reference
them in config (`scoreElementId`, `floatingTextIds`).

## 3. Create `src/sites/<id>/config.ts`

Copy `china-ex/config.ts` and adjust. Every field matters:

- `id`, `title`, `description`, `route` (`/<id>`)
- `svg` — `import svg from './map.svg?raw'`
- `levels` — value/label/color rows; colors must equal the SVG's `level=` fills
- `storageKey` — **unique** (e.g. `'<id>-levels'`), or sites clobber each other
- `regionGroupId`, `scoreElementId` — the SVG ids from step 2
- `floatingTextIds` — the title + score ids (they float on the export bg and
  flip color with the theme; get these right or text is illegible on dark bg)
- `coverSrc` — `'/cover/<id>.webp'` (created in step 6)
- `canvas` — usually `{ w: 1134, h: 976, ratio: 2 }` (match the SVG `viewBox`)
- `credit` — `{ author, url }` of the original project

## 4. Create `src/routes/<id>.tsx`

Copy `china-ex.tsx` verbatim, swapping the imported config and the
`createFileRoute('/<id>')` path. The route tree (`src/routeTree.gen.ts`) is
auto-generated by the TanStack plugin on the next `pnpm dev`/`pnpm build` — don't
hand-edit it.

## 5. Wire it up (two edits — don't skip the second)

- `src/sites/index.ts` — add the config to `SITES`. This alone drives the
  landing-page card **and** the sitemap (both iterate `SITES`).
- `vite.config.ts` — add `{ path: '/<id>' }` to the `tanstackStart` `pages`
  array. This is the easy-to-miss one: it's the explicit prerender list and is
  **not** derived from `SITES`. Without it the route isn't prerendered.
- `README.md` — add a row to the 页面 table.

## 6. Generate the OG cover → `public/cover/<id>.webp`

`coverSrc` is the social/og:image (the landing thumbnail renders the live SVG
instead, so the cover only shows up in link previews). Match the china-ex card:
2560×1280, `brandBg` background, the clean map (boxes only) on the left, title +
description on the right in `ZCOOLKuaiLe`. Covers are **lossless WebP** — for
these flat-color/sharp-edge cards that's ~5× smaller than PNG with no quality
loss (and smaller than JPEG, which also softens the black borders).

Use the bundled script — it embeds the font, hides labels/title/score/legend,
auto-colors a spread of regions, renders via headless Chrome (macOS), and
converts to lossless WebP with ImageMagick (`magick`):

```bash
node .claude/skills/add-ex-site/scripts/build-cover.mjs \
  "$(pwd)" <id> "<标题>" "<描述>"
```

Open the output and sanity-check it (font loaded, title on one line, map
balanced). If the title wraps, shorten it or tweak the script's font-size. On
non-mac, set `CHROME=/path/to/chrome` or render the generated HTML yourself.
Requires ImageMagick for the WebP conversion.

## 7. Verify (don't claim done without this)

```bash
pnpm build      # regenerates routeTree.gen.ts + prerenders /<id> (watch the log)
pnpm typecheck
pnpm lint       # expect only pre-existing warnings, none in new files
pnpm test
```

Then run it for real — `pnpm dev`, open `/<id>`, and confirm: the map renders
with labels, clicking a region (try one that was a converted `<rect>`) opens the
picker and colors it, the score updates, and `保存成图片` produces a correct
preview (title + score legible on the export background). Check the landing page
shows the new card and `/sitemap.xml` lists the route.
